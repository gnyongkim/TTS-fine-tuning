# 03. 데이터셋 준비 (KSS 베이스라인)

작성일: 2026-05-11
대상: GPT-SoVITS v2 한국어 파인튜닝 1차 베이스라인

---

## 1. 사용 데이터셋 — KSS

| 항목 | 값 |
|---|---|
| 이름 | Korean Single Speaker Speech Dataset (KSS) |
| 화자 | 여성 1인, 표준어 |
| 분량 | 12,853 클립 / 약 12.6시간 |
| 원본 포맷 | 44.1kHz, 16-bit PCM, mono WAV |
| 텍스트 | 원문 / 숫자 풀어쓰기 / 자모 분해 / 영어 번역 / 길이 |
| 출처 | Kyubyong Park, 2018 ([Kaggle](https://www.kaggle.com/datasets/bryanpark/korean-single-speaker-speech-dataset)) |
| HF 미러 | [`Bingsu/KSS_Dataset`](https://huggingface.co/datasets/Bingsu/KSS_Dataset) (3.81GB) |
| **라이선스** | **CC BY-NC-SA 4.0 — 비상업 전용** |

### 1.1 라이선스 주의 사항
KSS는 **비상업적 용도(연구·학습 검증)에 한해 사용 가능**하며, 상업 제품에 포함되는
모델 학습 데이터로 직접 사용할 수 없습니다.

본 프로젝트의 사용 방침은 다음과 같습니다.

1. **1단계 (현재) — 베이스라인 검증**
   KSS로 GPT-SoVITS v2 파인튜닝 파이프라인을 통째로 검증하고, 한국어 합성
   품질·학습 시간·VRAM 사용량을 측정합니다.
2. **2단계 (상용 직전) — 라이선스 호환 데이터셋으로 재학습**
   AIHub 한국어 음성 합성용 데이터셋(상용 가능 라이선스 항목) 또는 자체 녹음
   데이터로 동일 파이프라인을 재학습한 모델만 모바일 앱에 탑재합니다.

> 따라서 KSS로 만든 체크포인트(`*.ckpt`, `*.pth`, `*.onnx`)는 **반드시 사내 검증용으로만** 사용하고, 외부 배포·앱 번들 포함은 금지합니다.

## 2. 워크플로 (한 줄로 끝)

전처리 스크립트 [`scripts/prepare_kss.py`](../scripts/prepare_kss.py) 가 다운로드부터
GPT-SoVITS 학습 형식 생성까지 모두 처리합니다.

```bash
# 학습 서버에서
conda activate gptsovits
cd ~/projects/tts-finetune          # git clone 한 본 저장소
python scripts/prepare_kss.py --out-dir ~/data/kss_processed
```

스크립트가 수행하는 일:

1. HuggingFace `Bingsu/KSS_Dataset` 자동 다운로드 (인증 불필요, ~3.81GB)
2. 길이 필터 — 기본 3 ~ 10초 (GPT-SoVITS 권장)
3. 32kHz mono WAV로 리샘플 후 `~/data/kss_processed/wavs/` 저장
4. `expanded_script` 사용 (예: "1234" → "천이백삼십사")
5. 95:5 비율로 train/val 분할
6. 결과: `train.list`, `val.list`, `dataset_info.txt`

처리 시간은 RTX 4070 Ti 서버에서 디스크 I/O 기준 약 5~10분 (다운로드 시간 별도).

## 3. 출력물 구조

```
~/data/kss_processed/
├── wavs/
│   ├── kss_00000.wav          (32kHz mono, 16-bit PCM)
│   ├── kss_00001.wav
│   └── ... (필터 후 약 10,000~12,000개)
├── train.list                 (~95%)
├── val.list                   (~5%)
└── dataset_info.txt           (메타 요약)
```

`.list` 파일은 GPT-SoVITS v2의 표준 학습 입력 형식입니다.

```
/home/user/data/kss_processed/wavs/kss_00000.wav|kss|ko|그는 괜찮은 척하려고 애쓰는 것 같았다.
```

각 줄: `절대경로|화자명|언어코드|텍스트` (구분자 = `|`)

## 4. 옵션

`prepare_kss.py` 주요 인자:

| 옵션 | 기본값 | 설명 |
|---|---|---|
| `--out-dir` | `~/data/kss_processed` | 출력 폴더 |
| `--source` | `hf` | `hf`(HuggingFace) 또는 `local`(이미 받아둔 폴더) |
| `--src-dir` | — | `--source local` 일 때 KSS 압축 해제 폴더 |
| `--speaker` | `kss` | `.list` 의 화자 이름 |
| `--sr` | `32000` | 출력 샘플링 레이트 (Hz) |
| `--min-dur` | `3.0` | 최소 클립 길이 (초) |
| `--max-dur` | `10.0` | 최대 클립 길이 (초) |
| `--val-ratio` | `0.05` | 검증셋 비율 |
| `--text-field` | `expanded` | `expanded`(권장) / `original` / `decomposed` |
| `--limit` | `None` | 처리 개수 제한 (디버그) |
| `--dry-run` | `False` | 통계만 출력, 파일 저장 X |

빠른 검증용 (200개만 처리):

```bash
python scripts/prepare_kss.py --limit 200 --out-dir ~/data/kss_smoke
```

## 5. 대안 다운로드 경로

`Bingsu/KSS_Dataset` (HF) 가 막히는 경우 다음을 사용한 뒤 `--source local` 로 변환합니다.

### 5.1 Kaggle CLI

```bash
# 사전: ~/.kaggle/kaggle.json 에 본인의 Kaggle API 키 저장 (퍼미션 600)
pip install kaggle
mkdir -p ~/data/kss_raw && cd ~/data/kss_raw
kaggle datasets download -d bryanpark/korean-single-speaker-speech-dataset
unzip -q korean-single-speaker-speech-dataset.zip

# 변환
python scripts/prepare_kss.py --source local --src-dir ~/data/kss_raw \
    --out-dir ~/data/kss_processed
```

### 5.2 직접 다운로드
공식/미러 페이지에서 ZIP 다운로드 후 압축 해제 → 위와 동일하게 `--source local` 사용.
폴더에 `transcript.v.1.4.txt` 와 숫자 폴더(`1/ 2/ 3/ 4/`) 가 들어 있어야 합니다.

## 7. 검증 체크리스트

전처리 후 학습 전에 확인:

- [ ] `dataset_info.txt` 의 `total_clips` 가 11,000~12,500 범위 (필터 후 일반적인 KSS 수치)
- [ ] `total_minutes` 가 600~720분 사이
- [ ] `wavs/` 의 임의 파일을 `soundfile`로 열어 32000Hz / mono / int16 인지 확인
- [ ] `train.list` / `val.list` 합계 == `wavs/` 파일 개수
- [ ] `.list` 한 줄을 골라 한글 텍스트가 깨지지 않았는지 확인 (UTF-8)

빠른 검증 명령:

```bash
python - <<'PY'
import soundfile as sf, random
from pathlib import Path
out = Path.home()/"data/kss_processed"
lines = (out/"train.list").read_text(encoding="utf-8").strip().splitlines()
print(f"train lines: {len(lines)}")
sample = random.choice(lines)
wav, spk, lang, text = sample.split("|")
print(f"  sample : {wav}")
print(f"  speaker: {spk} | lang: {lang}")
print(f"  text   : {text}")
info = sf.info(wav)
print(f"  audio  : {info.samplerate}Hz, {info.channels}ch, {info.subtype}, {info.duration:.2f}s")
PY
```
