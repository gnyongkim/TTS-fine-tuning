# GPT-SoVITS v2 설치 가이드 (RTX 4070 Ti / Linux)

작성일: 2026-05-11
대상 환경: Ubuntu 22.04 LTS + NVIDIA RTX 4070 Ti (12GB) + CUDA 12.x

---

## 1. 설치 개요

| 항목 | 값 |
|---|---|
| Repo | `https://github.com/RVC-Boss/GPT-SoVITS` |
| Conda 환경명 | `gptsovits` |
| Python | 3.10 |
| PyTorch | 2.x (cu128 wheel, RTX 4070 Ti = sm_89) |
| 설치 경로 | `~/projects/GPT-SoVITS` (변경 가능) |
| 디스크 | 약 15~20GB (pretrained 포함) |
| VRAM | 학습 6~8GB, 추론 3~4GB |

설치 흐름은 공식 `install.sh`를 활용합니다. 공식 스크립트가
**PyTorch wheel(cu128) + 한국어 G2P 의존성 + pretrained 모델 + G2PW + NLTK + Open JTalk 사전**까지
한 번에 처리하므로, 본 프로젝트의 `setup.sh`는 환경 생성·사전 점검·검증을 감싸는 wrapper 역할을 합니다.

---

## 2. 사전 요구사항 (서버에서 미리 확인)

```bash
# (1) NVIDIA 드라이버 (>=535 권장)
nvidia-smi

# (2) Conda 설치 여부
conda --version
# 미설치 시:
#   wget https://repo.anaconda.com/miniconda/Miniconda3-latest-Linux-x86_64.sh -O /tmp/mc.sh
#   bash /tmp/mc.sh -b -p $HOME/miniconda3
#   $HOME/miniconda3/bin/conda init bash && exec bash

# (3) git, wget, unzip
sudo apt-get update
sudo apt-get install -y git wget unzip build-essential
```

> RTX 4070 Ti는 Ada Lovelace(Compute Capability 8.9)로,
> CUDA 12.6 / 12.8 빌드 모두 호환됩니다. 본 가이드는 **CU128(CUDA 12.8)** 권장.

## 3. 설치 실행 (한 줄)

본 프로젝트 폴더에 들어 있는 `setup.sh`와 `verify.py`를 서버로 복사한 뒤 실행합니다.

```bash
# 0. 파일 복사 (예: 로컬 → 서버)
#    scp setup.sh verify.py user@서버:~/

# 1. 권한 부여
chmod +x setup.sh

# 2. 실행 (기본값: CU128 + HuggingFace, 설치 위치 ~/projects)
bash setup.sh
```

옵션 예시:

```bash
# CUDA 12.6 wheel 사용
bash setup.sh --device CU126

# HuggingFace 다운로드가 막힌 경우 ModelScope에서 받기
bash setup.sh --source ModelScope

# UVR5(보컬 분리) 모델까지 같이 받기
bash setup.sh --uvr5

# 설치 위치/환경명 변경
bash setup.sh --install-dir /data/work --env-name gsv2
```

내부 진행 단계 (총 5단계):

1. **사전 점검** — OS, GPU, 드라이버, 디스크, conda, git/wget 확인
2. **conda 환경 생성** — `python=3.10`, 환경명 `gptsovits`
3. **저장소 clone** — `~/projects/GPT-SoVITS`
4. **공식 install.sh 실행** — PyTorch(cu128) + requirements.txt + pretrained + G2PW + NLTK + JTalk 사전 자동 다운로드
5. **CUDA 검증** — `torch.cuda.is_available()` / GPU 이름·VRAM 출력

## 4. 설치 후 검증

```bash
conda activate gptsovits
python ~/verify.py
```

`verify.py`는 다음을 검사합니다.

- Python 버전 (3.10/3.11)
- PyTorch + CUDA 사용 가능 여부, GPU 이름, sm_89 인식, VRAM
- CUDA matmul 실연산 테스트 (2048×2048 행렬곱)
- 핵심 의존성 (`transformers`, `gradio`, `librosa`, `funasr`, …)
- **한국어 의존성** (`g2pk2`, `ko_pron`, `python_mecab_ko`, `pyopenjtalk`)
- ffmpeg CLI
- `pretrained_models` / `G2PWModel` / `pretrained_models/sv` 폴더 존재 확인

모든 항목이 `[ OK ]` 로 표시되면 다음 단계(데이터셋 전처리)로 넘어가도 됩니다.

## 6. 자주 발생하는 이슈

| 증상 | 원인 / 해결 |
|---|---|
| `conda: command not found` | Miniconda 미설치. 위 사전 요구사항(2) 참고. |
| `nvidia-smi` 정상이지만 `torch.cuda.is_available() = False` | PyTorch가 CPU wheel로 잘못 설치됨. `pip install --force-reinstall torch torchcodec --index-url https://download.pytorch.org/whl/cu128` |
| install.sh 중간에 다운로드 실패 (HF 차단) | `bash setup.sh --source ModelScope` 로 재실행 |
| `pyopenjtalk` 빌드 실패 | `sudo apt-get install -y cmake build-essential` 후 재시도 |
| `mecab` import 실패 | `pip install python_mecab_ko` 재설치, 또는 `sudo apt install mecab libmecab-dev mecab-ipadic-utf8` |
| `RuntimeError: CUDA out of memory` | WebUI에서 batch size를 낮추거나, fp16 옵션 활성화 |
| `Permission denied: install.sh` | `chmod +x install.sh` 또는 `bash install.sh ...` 사용 |
