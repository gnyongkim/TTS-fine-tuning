## 3. 최종 추천: **GPT-SoVITS v2 (한국어 파인튜닝)**

### 3.1 추천 사유
1. **요구사항 적합도 1위** - LLM-style(AR-GPT) 구조 + 모바일 친화적 사이즈 + 공식 한국어 지원의 세 조건을 모두 충족하는 거의 유일한 모델.
2. **RTX 4070 Ti(12GB)에 최적** - 파인튜닝 시 약 6~8GB VRAM만 사용. 배치 사이즈 12까지 안정적.
3. **공개 데이터셋과의 궁합** - KSS(약 12시간), AIHub 감정·다화자 데이터셋과 호환되는 전처리 파이프라인이 공식 제공됨.
4. **ONNX 변환 검증** - GPT 부분(`s1`)과 SoVITS 부분(`s2`)을 각각 ONNX로 export하는 공식/커뮤니티 스크립트가 존재(`onnx_export.py`). React Native·Flutter 모바일 데모 사례 다수.
5. **Few-shot 능력** - 본 데이터셋이 부족하더라도 5초~1분 reference로 보조 화자 확장이 가능 → 향후 다화자 확장 시 재학습 부담 ↓.
6. **양자화 후 모바일 사이즈** - INT8 양자화 시 GPT ~80MB, SoVITS ~50MB → 합산 약 130MB로 균형형(100~300MB) 목표 달성.

### 3.2 모델 구성
- **GPT (s1)**: 텍스트 → semantic token (LLM-style autoregressive)
- **SoVITS (s2)**: semantic token + reference embedding → waveform (VITS 계열 디코더)
- **Reference Encoder**: 화자 임베딩 추출

### 3.3 학습/배포 파이프라인 개요
```
[텍스트+오디오 데이터셋]
        ↓
  전처리 (Whisper ASR 정렬, BERT 텍스트 인코딩, HuBERT semantic)
        ↓
  GPT (s1) 파인튜닝   ──┐
  SoVITS (s2) 파인튜닝 ──┴──> 통합 추론 검증
        ↓
  ONNX Export (s1.onnx, s2.onnx, vits.onnx 분리)
        ↓
  ONNX Runtime Mobile + INT8 양자화
        ↓
  Android (NNAPI) / iOS (CoreML EP) 배포
```

## 4. 백업 옵션 (상황별)

### 4.1 더 높은 품질이 필요하고 모바일 사이즈에 여유가 있을 때
- **XTTS-v2 한국어 파인튜닝 + INT8 양자화**
- 자연스러움/감정 표현이 GPT-SoVITS보다 한 수 위
- 단점: 양자화 후에도 600MB+ → 저사양 폰 제외해야 함

### 4.2 다화자/감정 확장을 강하게 원할 때
- **Spark-TTS 또는 Fish-Speech v1.5 한국어 파인튜닝**
- LLM 기반 controllability 우수
- 단점: 한국어 검증 사례 부족, ONNX 변환 안정성 미확인

### 4.3 ONNX 안정성 + 초경량을 원할 때 (LLM-style 포기 가능 시)
- **VITS2 / Piper 한국어 파인튜닝**
- 50MB 이하, ONNX 변환 매우 안정적
- 단점: LLM-style이 아니므로 본 프로젝트 요구사항 미충족

## 5. 다음 단계 권장 액션

1. GPT-SoVITS v2 공식 저장소(`RVC-Boss/GPT-SoVITS`) 클론 및 의존성 설치 (Python 3.10, PyTorch 2.x + CUDA 12.x)
2. 공개 한국어 데이터셋 1차 선정 (KSS 우선, 이후 AIHub 다화자로 확장)
3. 전처리 파이프라인 구축 (16kHz 다운샘플링, 무음 트림, 텍스트 정규화)
4. 베이스라인 파인튜닝 (GPT 8 epoch, SoVITS 8 epoch)으로 품질 목표 도달 여부 검증
5. ONNX export → INT8 양자화 → Android 시뮬레이터 RTF(Real-Time Factor) 측정
6. 목표 미달 시 백업 옵션(XTTS-v2)으로 전환 결정

## 6. GPT-SoVITS 버전 선정 — 왜 v2인가 (v3 / v4 / v2Pro 비교)

GPT-SoVITS 저장소에는 v1·v2·v2Pro·v3·v4가 공존합니다. 본 프로젝트는 **v2**를
선택했으며, 그 근거는 다음과 같습니다.

### 6.1 버전별 핵심 차이 (공식 README · Changelog 기준)

| 버전 | 출시 | 디코더 | 파인튜닝 VRAM | 학습 철학 | ONNX/모바일 |
|---|---|---|---|---|---|
| **v2** | 2024 | VITS 계열 (GAN) | **6~8 GB** | 학습셋 전체 음색 학습 | 사례 풍부, 안정 |
| v2Pro | 2025 | VITS 계열 개선 | v2 보다 약간 ↑ | v2와 동일 (학습셋 음색) | v2와 호환, 검증 진행 중 |
| v3 | 2025.02 | Flow Matching + BigVGAN | **14 GB** (LoRA 8 GB) | reference audio 추종 | 사례 적음, Flow Matching 반복 추론 |
| v4 | 2025.04 | v3 + 24→48 kHz SR | v3 와 유사 | v3 와 동일 | TorchScript export 지원, ONNX는 미검증 |

> 공식 README 인용 (영문 의역): *"v1/v2/v2Pro는 학습셋 전체의 음색을 학습하지만, v3/v4는
> reference audio 쪽으로 더 치우치며 학습셋 평균 품질에 더 민감합니다."*

### 6.2 본 프로젝트 제약 조건과의 매칭

| 제약 | v2 | v3 | v4 |
|---|---|---|---|
| **RTX 4070 Ti 12GB 풀 파인튜닝** | ✅ 6~8GB 사용 | ❌ 14GB 필요 (LoRA만 가능) | ❌ v3와 유사 |
| **모바일 모델 사이즈 100~300MB** | ✅ INT8 후 ≈130MB | △ BigVGAN 포함 시 부담 | △ 24/48kHz SR로 추가 부담 |
| **ONNX export 안정성** | ✅ 커뮤니티 사례 2년+ | △ Flow Matching 반복 추론 | △ TorchScript 우선, ONNX 미검증 |
| **단일 화자 안정적 고품질** | ✅ 학습셋 음색 학습 | △ reference 의존 | △ reference 의존 |
| **추론 RTF (모바일)** | ✅ 1-pass | △ Flow Matching 다단계 | △ + 슈퍼레졸루션 단계 |

### 6.3 결론

세 가지 제약(**12GB VRAM 풀 파인튜닝 가능 / 모바일 사이즈 100~300MB / ONNX 모바일 검증**)을
모두 만족하는 버전은 v2가 유일합니다. v3·v4의 강점인 zero-shot voice cloning은 본 프로젝트의
목표(앱에 탑재할 안정적 음색)와도 일치하지 않습니다.

### 6.4 향후 업그레이드 경로 — v2Pro

2025년 추가된 v2Pro는 *"v2의 하드웨어 비용·속도를 유지하면서 v4 수준 품질"* 을 목표로
설계된 변형판입니다. 저장소에 `s2v2Pro.json` config 가 이미 포함되어 있으며 학습 코드
(`s2_train.py`)는 v2와 공유하므로 본 저장소 파이프라인을 거의 그대로 둔 채 config·
pretrained 가중치만 교체해 업그레이드 실험이 가능합니다.

**권장 2단계 로드맵:**
1. **현재** — v2 베이스라인 학습 완료 → 품질·RTF 기준점 확보 → ONNX/모바일 검증
2. **다음** — 동일 데이터로 v2Pro 재학습 후 A/B 비교 → 품질 향상 폭이 의미 있고
   ONNX 변환·모바일 RTF가 통과하면 v2Pro 로 전환

v3·v4는 현 제약 조건에서는 후보에서 제외합니다.

(v3·v4 는 12GB VRAM 풀 파인튜닝 불가 + 모바일 ONNX 미검증 / v2Pro 는 2차 업그레이드 후보)
