# 04. GPT-SoVITS v2 파인튜닝 가이드 (KSS 베이스라인)

작성일: 2026-05-12
대상: NVIDIA RTX 4070 Ti (12GB VRAM) / Linux / KSS 전처리 완료 상태

---

## 1. 단계 개요

GPT-SoVITS v2 는 두 단계 학습으로 구성됩니다.

| 단계 | 모델 | 입력 | 출력 | 역할 |
|---|---|---|---|---|
| s1 | **GPT (AR LM)** | 음소(BERT) → semantic token | text-to-semantic | "무엇을 말할지" |
| s2 | **SoVITS Decoder** | semantic token + spk embed → waveform | semantic-to-audio | "어떻게 들리게 할지" |

두 모델은 **독립적으로 학습**합니다. 단일 4070 Ti(12GB)에서는 동시 학습이 VRAM
부족으로 불가능하므로 순차 진행을 권장합니다 (s1 먼저 → s2). GPU가 2장 이상이면 동시 진행 가능.

## 4. 전처리: BERT / HuBERT / Semantic 토큰 추출

`prepare_kss.py` 가 만든 `.list` 파일을 GPT-SoVITS 학습 형식으로 변환합니다.

```bash
source configs/experiment.env
python scripts/preprocess_for_training.py
```

내부 동작 (3단계, GPT-SoVITS 의 `prepare_datasets/{1,2,3}-*.py` 호출):

1. **`1-get-text.py`** — 텍스트 → 음소 + BERT 임베딩 (`2-name2text.txt`, `3-bert/`)
2. **`2-get-hubert-wav32k.py`** — HuBERT SSL 임베딩 + 32kHz wav (`4-cnhubert/`, `5-wav32k/`)
3. **`3-get-semantic.py`** — semantic token (`6-name2semantic.tsv`)

산출물은 모두 `$EXP_DIR` 안에 생성됩니다.

| 산출물 | 크기 추정 | 용도 |
|---|---|---|
| `2-name2text.txt` | ~1MB | s1 학습 입력 |
| `3-bert/` | ~3GB (12k 클립) | s1 학습 입력 |
| `4-cnhubert/` | ~5GB | s2 학습 입력 |
| `5-wav32k/` | ~2.5GB | s2 학습 입력 |
| `6-name2semantic.tsv` | ~5MB | s1·s2 공통 |

처리 시간 (4070 Ti 기준): 약 30~60분.

> 재실행 시 일부 단계만 다시 돌리려면: `--skip text` `--skip hubert` `--skip semantic`

## 5. s1 (GPT) 학습

```bash
source configs/experiment.env
bash scripts/train_s1.sh
```

[`configs/s1_kss.yaml`](../configs/s1_kss.yaml) 의 4070 Ti 12GB 튜닝 값:

| 항목 | 값 | 비고 |
|---|---|---|
| `train.batch_size` | 8 | OOM 시 6 → 4 |
| `train.epochs` | 15 | KSS 단일화자 + pretrained 활용 시 충분 |
| `train.precision` | `16-mixed` | fp16 (4070 Ti 권장) |
| `train.save_every_n_epoch` | 5 | 디스크 절약 |
| `optimizer.lr` (peak) | 0.01 | 너무 빠르게 발산하면 0.005 |
| `optimizer.warmup_steps` | 2000 | |

**예상 학습 시간**: 약 6~10시간 (15 epoch, 12k 클립, batch 8)

산출물:
- `$EXP_DIR/logs_s1/` — TensorBoard 이벤트 + 원본 ckpt
- `$EXP_DIR/weights_s1/kss_baseline_s1-eN.ckpt` — 추론용 half-precision 가중치

모니터링:
```bash
# 로컬 Mac 에서 SSH 포트포워딩
ssh -L 6006:localhost:6006 user@서버
# 서버 (다른 셸):
tensorboard --logdir ~/data/kss_exp/kss_baseline/logs_s1 --port 6006
# Mac 브라우저: http://localhost:6006
```

체크 포인트:
- `train/loss` 가 epoch 5쯤 2~3 부근에서 안정화되면 정상
- `val/loss` 가 5 epoch 째부터 발산하면 lr 낮춤(0.01 → 0.005) 또는 epoch 단축

## 6. s2 (SoVITS) 학습

```bash
source configs/experiment.env
bash scripts/train_s2.sh
```

[`configs/s2_kss.json`](../configs/s2_kss.json) 의 4070 Ti 12GB 튜닝 값:

| 항목 | 기본 → 조정 | 비고 |
|---|---|---|
| `train.batch_size` | 32 → **6** | A100 80GB 기준 32. 4070 Ti 12GB는 6 이하 |
| `train.epochs` | 100 → **25** | KSS + pretrained로 충분 |
| `train.fp16_run` | true | 4070 Ti 필수 |
| `train.grad_ckpt` | false → **true** | gradient checkpointing(VRAM 절감, 약 30% 느림) |
| `train.save_every_epoch` | — → **4** | ckpt 약 6~7개 |
| `train.segment_size` | 20480 | OOM 시 16384 |

**예상 학습 시간**: 약 8~14시간 (25 epoch, 12k 클립, batch 6, grad_ckpt)

산출물:
- `$EXP_DIR/logs_s2/` — TensorBoard + `G_*.pth`/`D_*.pth` (Generator/Discriminator)
- `$EXP_DIR/weights_s2/kss_baseline_s2_eN_sM.pth` — 추론용

모니터링:
```bash
tensorboard --logdir ~/data/kss_exp/kss_baseline/logs_s2 --port 6007
```

체크 포인트:
- `loss/g/total` 안정화 + `loss/g/mel` 의 부드러운 감소
- 10 epoch 부근에서 합성 샘플(`eval/audio`) 들으며 자연스러움 확인

## 7. 권장 진행 순서 (4070 Ti 단일)

```
1. preprocess_for_training.py   (30~60 분)
2. train_s1.sh                  (6~10 시간)        ← tmux 세션 권장
3. train_s2.sh                  (8~14 시간)        ← s1 끝난 뒤 순차 진행
4. 추론 테스트 (다음 가이드)
─────────────────────────────────────
총: 약 1~1.5일
```

GPU 가 2장이면 s1·s2 를 다른 GPU에 동시 할당:
```bash
GPU_ID=0 EXP_NAME=kss_baseline bash scripts/train_s1.sh &
GPU_ID=1 EXP_NAME=kss_baseline bash scripts/train_s2.sh &
```

## 9. 주요 트러블슈팅

| 증상 | 해결 |
|---|---|
| `CUDA out of memory` (s1) | `configs/s1_kss.yaml` 의 `train.batch_size` 8 → 6 → 4 |
| `CUDA out of memory` (s2) | `configs/s2_kss.json` 의 `train.batch_size` 6 → 4 → 2, `segment_size` 20480 → 16384 |
| `pretrained s1 가 없습니다` | `setup.sh` 재실행 (HF 차단 시 `--source ModelScope`) |
| `inp_text 환경변수 없음` | `source configs/experiment.env` 빠뜨림 |
| `2-name2text.txt 없음` (s1 시작 시) | `preprocess_for_training.py` 미실행. 먼저 실행 |
| TensorBoard 접속 안 됨 | SSH 포트포워딩(`-L 6006:localhost:6006`) 누락 |
| 학습 도중 NaN | mixed precision 충돌. 임시로 `precision: 32-true` (s1) 또는 `fp16_run: false` (s2) |
| 너무 빠르게 수렴 / 발음 깨짐 | s1 lr 0.01 → 0.005, s1 epochs 15 → 10 |
